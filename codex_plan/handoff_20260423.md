# Handoff 2026-04-23: ValueNet 增加 episode_first_head_img

## 用户目标

执行 `bash 001train_torch_valuenet.sh` 训练 `value_pretrain_16dim` 时，在原有三路图像：

- `base_0_rgb`
- `left_wrist_0_rgb`

- 在 `restructure(traj)` 中用 episode 第一帧 head 图构造全 episode 对齐字段：

```python

    traj["observation"]["image_head"][0:1],
```

## 如何判断第 4 张图进入模型

严格说，当前日志没有直接打印：

```text
len(images)=4
image_keys=[...]
image shapes=[...]
```

所以它不是“日志直接显示第 4 张 tensor shape”的证明。

当前判断依据是“日志中的四路配置 + 代码必经路径 + 模型前向成功”：

1. `scripts/train_pytorch_valuenet.py` 用 `data_config.image_keys` 构造 `ValueNetPytorch`。
2. `ValueNetPytorch` 保存 `self.image_keys`。
3. `_preprocess_observation()` 调用 `preprocess_observation_pytorch(..., image_keys=self.image_keys)`。
4. `preprocess_observation_pytorch()` 会检查：

```python
if not set(image_keys).issubset(observation.images):
    raise ValueError(...)
```

5. 然后按 `image_keys` 逐个取图：

```python
for key in image_keys:
    image = observation.images[key]
```

6. 后续 `ValueNetPytorch.embed_prefix()` 对 `images` 中每张图执行：

```python
self.paligemma_with_expert.embed_image(img)
```

因此如果第 4 个 key 不存在，或者没有通过 transform 进入 `observation.images`，会在 preprocess 阶段直接报 `images dict missing keys`；实际日志显示训练已持续 forward/backward 到 283xx step。

如果后续需要 100% 日志级证明，建议临时加只打印一次的 debug，例如在 `ValueNetPytorch._preprocess_observation()` 里 rank0/step0 打印：

```python
print(f"[ValueNet image_keys] {self.image_keys}")
print(f"[ValueNet image count] {len(images)}")
print(f"[ValueNet image shapes] {[tuple(x.shape) for x in images]}")
```

不要在当前正在跑的训练上改代码；等下次重启训练前再加。
